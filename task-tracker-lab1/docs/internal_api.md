# Внутренний API

Проект содержит отдельный набор конечных точек, 
предназначенных только для внутренних сервисов.
Эти эндпоинты недоступны обычным клиентам приложения.

## Защита внутреннего API

Доступ осуществляется по специальному заголовку:

X-Internal-Key: <секретный ключ>

Если ключ:
- **не передан**  422 (некорректный запрос)
- **неверный**  403 Forbidden

Ключ задаётся через переменную окружения:

INTERNAL_API_KEY


## Доступные эндпоинты

### `GET /api/internal/stats`

Возвращает агрегированные данные системы:

{
  "total_users"
  "total_projects"
  "total_tasks"
  "total_comments"
}

## Зачем нужен внутренний API

 - мониторинг состояния системы
 - сбор статистики для админ-панели или аналитики
 - выполнение внутренних технических задач
 - не требует JWT и не зависит от пользователей

## Почему это внутренний эндпоинт

 - использует отдельный ключ доступа
 - не фильтрует данные по пользователю
 - возвращает служебную статистику
 - не предназначен для внешних клиентов
