# Пагинация (limit / offset)

API поддерживает постраничную выборку данных для всех эндпоинтов,
возвращающих списки.

## Параметры пагинации

Используется классическая SQL-модель:

?limit=<количество>
?offset=<смещение>

### По умолчанию:
- `limit = 10`
- `offset = 0`

## Где применяется

Пагинация включена в:

- `/api/v1/projects`
- `/api/v1/projects/{project_id}/tasks`
- `/api/v1/tasks/{task_id}/comments`
- `/api/v2/projects/{project_id}/tasks`

## Зачем это нужно
- уменьшение нагрузки на сервер  
- ускорение ответа  
- обработка больших коллекций  
- стандартная практика REST API
