# Идемпотентность POST-запросов

Для предотвращения дублирования данных и повторного выполнения операций  
API поддерживает механизм идемпотентности через заголовок:

Idempotency-Key: <уникальное значение>

## Принцип работы
1. Клиент отправляет POST-запрос с уникальным ключом.  
2. Сервер сохраняет результат выполнения под этим ключом.  
3. Повторный запрос с тем же ключом:
   - не создаёт новый ресурс,
   - возвращает тот же ответ, что и первый запрос.

## Где используется
Идемпотентность включена для операций:
- создание проектов  
- создание задач (v1 и v2)  
- добавление комментариев  

## Зачем это нужно
- безопасная работа при повторных запросах (сети, перезагрузки, таймауты)  
- защита от случайных дублей  
- корректная реализация REST-паттернов
