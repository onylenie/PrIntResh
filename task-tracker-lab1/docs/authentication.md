## Выбранная модель безопасности
В проекте используется **JWT-аутентификация** с двумя типами токенов:
- **Access Token** — 15 минут
- **Refresh Token** — 7 дней

### Почему выбран JWT
- Сервер не хранит сессии, токен самодостаточен, API легко масштабировать.
- Подходит для SPA, мобильных приложений и микросервисов.
- Позволяет применять короткий срок жизни access-токена для безопасности.
- Refresh-токен позволяет обновлять access-токен без повторного логина.
